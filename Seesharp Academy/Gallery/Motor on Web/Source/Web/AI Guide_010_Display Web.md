## 总体要求
写一个网页motor_monitor.html，目的是大屏幕展示一个电机的工作状态，通过本机Redis服务接收数据
## 屏幕内容：
题头：“锐视公学电机状态监测”
前景区域：
* 中部上2/3面积显示相对目录model下的“船桨模型.glb”，加渲染允许鼠标操作旋转放大，没有鼠标操作时缓慢旋转展示。模型用容器承载，旋转容器，使得旋转轴在中心位置。
* 中部下1/3面积添加一个即时频率的柱状图，横坐标是频率，纵坐标是幅度。 显示频率范围和频谱范围相同。注意显示内容是即时频率及幅度。 柱子的颜色和幅度相关渲染。 这个图形控件可能不是严格的柱状图。Y轴用 0.0e0格式。柱状图的柱子宽度固定为4像素。
* 左侧显示曲线图线
  * 从上到下排布：时域信号、频谱、即时频率历史strip chart
  * 即时频率历史横坐标为时间，纵坐标为频率，每个即时频率读数显示一个点，每读取一次频率列表刷新一列。
* 右侧显示即时频率及其幅度表格
  * 表格第一列为序号，第二列为频率，第三列为幅度
* 曲线和表格都以Redis数据刷新而更新，但是更新间隔不低于500ms
* 如果存在大片空白，用高科技感素材填充
背景：暗夜星空的深蓝色基调，有些不影响视觉注意力的星辰或装饰
## Redis数据库：
* 变量列表，按照功能：名称（引号内），数据类型描述编写
  * 时域信号： "SeesharpMotor:TimeWaveform": 逗号分割的浮点数
  * 时域信号采样间隔时间(s)："SeesharpMotor:Waveform_dT": 浮点数
  * 信号频谱："SeesharpMotor:Spectrum": 逗号分割的浮点数
  * 频谱采样间隔频率(Hz)："SeesharpMotor:Spectrum_dF": 浮点数
  * 即时频率："SeesharpMotor:Frequencies": 逗号分割的浮点数
  * 即时频率幅度："SeesharpMotor:Amplitudes": 逗号分割的浮点数,与即时频率一一对应，需容忍变量刷新竞争带来的瞬时不匹配
## 测试程序
在Test目录建立一个Python测试程序，模拟一个电机噪声信号，及其即时频率（主频和谐波）发送给Redis。
