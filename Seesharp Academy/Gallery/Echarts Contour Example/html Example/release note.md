# ECharts Contour Chart Example — Release Note

## Overview

A self-contained ECharts 6 contour chart demo using `@echarts-x/custom-contour` and D3 v7 density estimation.

Includes 4 examples:

1. Single-cluster random density contour
2. Bimodal Gaussian distribution contour
3. Line-only mode (no fill)
4. High-resolution heatmap contour with visualMap

## Prerequisites

- **Node.js >= 12** (includes npm) — download from [https://nodejs.org](https://nodejs.org)
- A modern browser (Chrome / Edge / Firefox)
- Internet connection (one-time, for `npm install` only)

## Installation Steps (Windows)

### Step 1: Install Node.js

1. Go to [https://nodejs.org](https://nodejs.org)
2. Download the **LTS** version (recommended)
3. Run the installer with default settings
4. Verify installation — open PowerShell and run:

```powershell
node --version
npm --version
```

Both commands should print a version number (e.g., `v20.x.x` and `10.x.x`).

### Step 2: Open PowerShell and Navigate to the Source Folder

```powershell
cd "D:\path\to\source"
```

> Replace with the actual path where you extracted this package.  
> The folder must contain `package.json`.

### Step 3: Install Dependencies

```powershell
npm install
```

This will:
- Read `package.json` for declared dependencies
- Read `package-lock.json` for exact pinned versions
- Download packages from `registry.npmjs.org` into a local `node_modules/` folder
- Total download size: ~4 MB

Wait until you see a message like:

```
added 35 packages in 5s
```

### Step 4: Open the Page

```powershell
start index.html
```

Or simply double-click `index.html` in File Explorer. The page opens in your default browser — no server needed.

## Offline / Air-Gapped Deployment

If the target machine has **no internet access**:

1. On a connected machine, complete Steps 1–3 above
2. Copy the **entire folder** (including the generated `node_modules/`) to the target machine via USB or shared drive
3. Open `index.html` directly — no network required at runtime

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `'npm' is not recognized` | Node.js not installed or not in PATH. Reinstall Node.js and restart PowerShell. |
| `npm install` fails with network error | Check internet/proxy settings. Try `npm config set registry https://registry.npmmirror.com` for China mirror. |
| Page shows blank charts | Ensure `npm install` completed successfully and `node_modules/` exists in the same folder as `index.html`. |
| `EACCES` or permission errors | Run PowerShell as Administrator, or fix folder permissions. |

## Notes

- No build step, no bundler, no dev server required
- After `npm install`, the page runs **100% offline** — zero CDN, zero external requests
- Load order matters: `echarts.js` → `d3.js` → `contour.auto.js` (already handled in `index.html`)
