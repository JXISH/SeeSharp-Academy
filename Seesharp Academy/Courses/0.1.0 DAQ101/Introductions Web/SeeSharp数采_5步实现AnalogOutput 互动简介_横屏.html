1创建任务 · Create Task JY5500AOTask

模拟输出任务 (AO Task) 是一个管理硬件输出的工作对象 — 它把"硬件 + 通道 + 输出参数 + 触发"全部封装在一起，负责将数字信号转换为模拟电压输出。

类比 Analogy： 你坐在 5510 板卡前，打算给设备输出一段特定波形。先按下电源、登记座位号 (Slot Number)，于是诞生了一个 aoTask 对象。它知道自己要操作哪一张板卡，但还不知道具体从哪一路输出、用什么更新率。

2添加通道 · AddChannel

        
告诉任务"我要从哪一路输出信号、输出量程多大"。


        

          关键概念 Key Idea： AddChannel 把物理引脚 (AO0~AO3) 映射到任务里。一旦添加，硬件就为该通道分配 DAC (数模转换器) 与输出缓冲区。
        


        

          
参数 Parameter说明 Description取值 Range

          
            
chnId
通道号 Channel ID		
              物理引脚序号5510/5515: 0–3 · 5511/5516: 0–1

            
rangeLow / rangeHigh
量程下/上限 Range		
              输出电压范围
              ±10V / ±5V

          
        


        

          📌 量程选择提示 Range Tip： JY5510 AO 支持 ±10V 和 ±5V 两种量程。选择 ±5V 时，16-bit DAC 的分辨率约为 0.15 mV/step；选择 ±10V 时约为 0.3 mV/step。使用较小量程可获得更高输出精度。

3配置参数 · Configure

        
设定"输出模式 + 更新率 + 输出点数"，决定波形将以什么速度、输出多少点、如何循环。


        

          更新率 UpdateRate (Sa/s)： 每秒钟输出几个点。更新率越高，输出波形越平滑。例如输出 1 kHz 正弦波，若更新率为 100 kSa/s，则每周期有 100 个点，波形非常光滑。
        


        

          输出模式 AOMode 四种：
          
    
            
  • Single 单点 — 直接写入 DAC 寄存器，立即输出一个电压值
    • 
            
  • Finite 有限 — 输出固定 N 点后自动停止
    • 
            
  • ContinuousWrapping 连续循环 — 循环输出固定缓冲区中的波形 (本示例使用)
    • 
            
  • ContinuousNoWrapping 连续非循环 — 持续消耗缓冲区数据，可实时追加新数据
    • 
          

        


        

          
属性 Property类型 Type说明 Description

          
            
ModeAOMode输出模式 / Output mode

            
UpdateRatedouble每通道更新率 / Sa/s per channel

            
SamplesToUpdateintFinite 模式总输出点数 / Finite-mode samples per channel

4写入数据 · Write Data

        
将波形数据写入输出缓冲区。对于 ContinuousWrapping 模式，必须在 Start() 之前调用 WriteData。


        

          缓冲区 (Buffer) 模型：
          AO 缓冲区存储待输出的波形数据。ContinuousWrapping 模式下硬件会循环读取缓冲区中的数据并输出；ContinuousNoWrapping 模式下数据输出后不重复，需要应用程序持续追加。
        


        

          
方法 / 属性说明

          
            
WriteData(double[] buf, int timeout)单通道写入波形数据 / Write waveform (single channel)

            
WriteData(double[,] buf, int timeout)多通道写入（列存储，rows=samples, cols=channels）

            
WriteSinglePoint(double value, int ch)Single 模式直接输出一个电压值

            
AvaliableLenInSamples缓冲区可写入的样本数 / Available buffer space

            
BufLenInSamples缓冲区总大小 / Total buffer size per channel

          
        


        

          ⚠️ 关键顺序 Key Order： ContinuousWrapping 模式必须 先 WriteData → 后 Start。否则硬件没有数据可输出，将抛出 BufferDataDownflow 异常。

5启动与停止 · Start & Stop

        
调用 Start() 启动输出，硬件按更新率将缓冲区数据转换为模拟电压。结束后 Stop() + Channels.Clear() 释放资源。


        

          触发 Trigger： Start() 之后等待"触发条件"满足才真正开始输出。
          
    
            
  • Immediate 立即触发 (默认) — Start 后马上输出
    • 
            
  • Digital 数字触发 — 等待 PFI 引脚的上升/下降沿
    • 
            
  • Soft 软触发 — 调用 SendSoftwareTrigger() 开始输出
    • 
          

        


        

          时钟同步 Clock Sync： AO 支持内部时钟和外部时钟。
          
    
            
  • Internal 内部时钟 (默认) — 使用板卡自身振荡器
    • 
            
  • External 外部时钟 — 接收外部信号 (如 AI_SampleClock / PFI) 作为更新时钟，实现 AO 与 AI 或其他设备同步输出
    • 
          

        


        

          ⚠️ 常见错误 Common Pitfall：
          
    
            
  • Start 之后再修改参数 → TaskHasStartedCannotPerformTheSetOperation
    • 
            
  • ContinuousWrapping 未先 WriteData 就 Start → BufferDataDownflow
    • 
            
  • 务必在结束后 Stop() + Channels.Clear()，避免下次启动残留
    • 
          

        


        

          
方法 / 属性 Member说明 Description

          
            
Start()启动输出 / Start the AO task

            
Stop()停止输出 / Stop the AO task

            
WaitUntilDone(timeout)Finite 模式阻塞等待完成 / Block until Finite done

            
Channels.Clear()清除所有通道 / Clear channels for reuse

            
Trigger.TypeAOTriggerType: Immediate / Digital / Soft

            
SampleClock.SourceAOSampleClockSource: Internal / External
