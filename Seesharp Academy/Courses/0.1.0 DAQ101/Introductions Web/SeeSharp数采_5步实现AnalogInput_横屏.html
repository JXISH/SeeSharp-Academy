1创建任务 · Create Task JY5500AITask

采集任务 (Task) 是一个一次性的工作对象 — 它把"硬件 + 通道 + 参数 + 触发"全部封装在一起，仿佛在 PXIe 槽位上"租用"了一台采集仪。

类比 Analogy： 你坐在 5510 板卡前，先按下电源、登记座位号 (Slot Number)，于是诞生了一个 aiTask 对象。它知道自己要操作哪一张板卡，但还不知道具体测哪一路信号、用什么采样率。

2添加通道 · AddChannel

        
告诉任务"我要测哪一路信号、量程多大、用何种接线方式"。


        

          关键概念 Key Idea： AddChannel 把物理引脚 (AI0~AI31) 映射到任务里。一旦添加，硬件就开始为这一路信号分配增益放大器 (PGA) 与缓冲区。
        


        

          
参数 Parameter说明 Description取值 Range

              物理引脚序号；-1 表示全部5510: 0–31 (SE) / 0–7 (Diff)

量程下/上限 Range		
              选择略大于信号峰值的量程 → 精度最高
              ±10/±5/±2/±1/±0.5/±0.2/±0.1 V

            
AITerminal

3配置参数 · Configure

        
设定"采集模式 + 采样率 + 采集点数"，决定波形将以什么速度、采多少点、是否一次完成。


        

          采样率 SampleRate (Sa/s)： 每秒钟采几个点。Nyquist 准则：采样率 ≥ 2 × 信号最高频率。例如要看 10 kHz 正弦波，采样率必须 ≥ 20 kSa/s，工程上常用 5–10 倍以保留波形细节。
        


        

          采集模式 AIMode 四种：
          
    
            
  • Single 单点 — 软件触发一次读一个值，类似万用表
    • 
            
  • Finite 有限 — 采固定 N 点后自动停止 (本示例使用)
    • 
            
  • Continuous 连续 — 持续采集，应用程序轮询读取
    • 
            
  • Record 录制 — 数据流式落盘，支持预览
    • 
          

        


        

          
属性 Property类型 Type说明 Description

          
            
ModeAIMode采集模式 / Acquisition mode

            
SampleRatedouble每通道采样率 / Sa/s per channel

            
SamplesToAcquireintFinite 模式总点数 / Finite-mode samples per channel

4启动采集 · Start

        
参数配置完成后，调用 Start() — 硬件开始按设定采样率采样、写入板载缓冲区。


        

          触发 Trigger： Start() 之后并不一定立即采集，要等到"触发条件"满足。
          
    
            
  • Immediate 立即 (默认)
    • 
            
  • Digital 数字边沿 (PFI0 上升/下降沿)
    • 
            
  • Analog 模拟阈值 (例如 AI0 越过 2.5V)
    • 
            
  • Soft 软触发 (调用 SendSoftwareTrigger())
    • 
          

        


        

          ⚠️ 常见错误 Common Pitfall： Start 之后再修改 SampleRate / Mode 会抛出
          TaskHasStartedCannotPerformTheSetOperation 异常 — 必须先 Stop() 才能改参。
        


        

          
方法 / 属性 Member说明 Description

          
            
Start()启动任务，硬件开始采样 / Start the task

            
SendSoftwareTrigger()软件发送一次触发 / Issue a soft trigger

            
Trigger.Type触发类型枚举 / AITriggerType enum

5读取数据 · ReadData

        
数据已经在板卡的 DMA 缓冲区中。调用 ReadData 把它搬到 .NET 数组中，绘图或写文件。


        

          缓冲区 (Buffer) 模型：
          硬件持续以 SampleRate 写入环形缓冲区 (FIFO)；应用程序周期性轮询 AvailableSamples，达到 N 点后调用 ReadData(ref buf, N, timeout)。
          
读不及时 → BufferDataOverflow；读取速度过慢通常需要：① 提高读取频率，或 ② 降低采样率。
        


        

          
方法 / 属性说明

          
            
AvailableSamples当前缓冲区可读样本数 / Samples ready in buffer

            
ReadData(ref double[] buf, int n, int timeout)单通道读取，timeout=-1 永久等待

            
ReadData(ref double[,] buf, int n, int timeout)多通道读取，列存储 (rows=samples, cols=channels)

            
WaitUntilDone(timeout)Finite 模式阻塞等待全部点数完成

            
Stop() + Channels.Clear()采集完毕务必 Stop 并清通道，避免下次启动残留

          
        


        

          典型 Finite 流程 Pattern (来自 Winform_AI_Finite 范例)：
          
Timer Tick:
  if (aiTask.AvailableSamples >= readValue.Length) {
      aiTask.ReadData(ref readValue, readValue.Length, -1);
      easyChartX.Plot(readValue);
      aiTask.Stop();
      aiTask.Channels.Clear();
  }
