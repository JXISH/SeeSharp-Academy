1创建任务
Create Task · JY5500AITask

采集任务 (Task) 把"硬件 + 通道 + 参数 + 触发"封装成一个对象，相当于在 PXIe 槽位上"租用"一台采集仪。

类比：你坐在 5510 板卡前，先按下电源、登记座位号 (Slot Number)，于是诞生了一个 aiTask。它知道操作哪张板卡，但还不知道测哪一路、用什么采样率。

2添加通道
AddChannel

      
告诉任务"我要测哪一路、量程多大、用何种接线方式"。


      

        关键概念：AddChannel 把物理引脚映射到任务里。一旦添加，硬件就为这一路分配增益放大器 (PGA) 与缓冲区。
      


      

        

          
chnId · 通道号

          
物理引脚序号；-1 表示全部5510: 0–31 (SE) / 0–7 (Diff)

        

        

          
rangeLow / rangeHigh · 量程

          
选择略大于信号峰值 → 精度最高±10 / ±5 / ±2 / ±1 / ±0.5 / ±0.2 / ±0.1 V

        

        

          
AITerminal · 终端配置

          
RSE 单端 / NRSE 非参考单端 / Differential 差分Wiring Configuration

        

      


      

        📌 量程口诀：信号 ≤ 0.2V → 用 ±0.2V 量程，充分利用 18-bit 分辨率；信号超过 ±5V → 必选 ±10V，避免削顶 (clipping)。

3配置参数
Configure Parameters

      
设定"采集模式 + 采样率 + 采集点数"，决定波形以什么速度、采多少点、是否一次完成。


      

        采样率 SampleRate (Sa/s)：每秒采几个点。

        Nyquist 准则：采样率 ≥ 2 × 信号最高频率。看 10 kHz 正弦波，采样率必须 ≥ 20 kSa/s，工程上常用 5–10 倍。
      


      

        采集模式 AIMode 四种：
        
    
          
  • Single 单点 — 类似万用表
    • 
          
  • Finite 有限 — 采固定 N 点 (本例)
    • 
          
  • Continuous 连续 — 持续采集
    • 
          
  • Record 录制 — 流式落盘
    • 
        

      


      

        

          
Mode · AIMode

          
采集模式枚举Acquisition mode

        

        

          
SampleRate · double

          
每通道采样率Sa/s per channel

        

        

          
SamplesToAcquire · int

          
Finite 模式总点数Finite-mode samples per channel

4启动采集
Start

      
参数配置完毕，调用 Start()，硬件按设定采样率写入板载缓冲区。


      

        触发 Trigger：Start 后等触发条件满足才开始采集。
        
    
          
  • Immediate 立即 (默认)
    • 
          
  • Digital 数字边沿 (PFI0 上升/下降沿)
    • 
          
  • Analog 模拟阈值 (AI0 越过 2.5V)
    • 
          
  • Soft 软触发 (调用 SendSoftwareTrigger)
    • 
        

      


      

        ⚠️ 易错点：Start 后再改 SampleRate/Mode 会抛 TaskHasStartedCannotPerformTheSetOperation — 必须先 Stop 再改参。

5读取数据
ReadData

      
数据已在板卡 DMA 缓冲区。调用 ReadData 搬到 .NET 数组中绘图或写文件。


      

        缓冲区 Buffer 模型：

        硬件持续以 SampleRate 写入环形缓冲区 (FIFO)；应用周期性轮询 AvailableSamples，达到 N 点后调 ReadData。

        读不及时 → BufferDataOverflow，需提高读取频率或降低采样率。
      


      

        

          
AvailableSamples

          
缓冲区可读样本数Samples ready in buffer

        

        

          
ReadData(ref buf, n, timeout)

          
读到本地数组；timeout = -1 表示永久等待double[] 单通道 / double[,] 多通道列存储

        

        

          
WaitUntilDone(timeout)

          
Finite 模式阻塞等待全部点数完成Block until finite acquisition done

        

        

          
Stop() + Channels.Clear()

          
采集完毕务必 Stop 并清通道Avoid leftover channels next time

        

      


      

        典型 Finite 流程 (Winform_AI_Finite 范例)：

Timer Tick:
  if (aiTask.AvailableSamples
      >= readValue.Length) {
    aiTask.ReadData(ref readValue,
      readValue.Length, -1);
    chart.Plot(readValue);
    aiTask.Stop();
    aiTask.Channels.Clear();
  }
