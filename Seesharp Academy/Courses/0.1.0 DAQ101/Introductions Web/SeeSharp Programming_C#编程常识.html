2 软件结构 Software Structure

从一个最简单的 “Hello” 程序出发，认清每一行代码在做什么。

using System;                       // 引用 / Using

namespace HelloApp                    // 命名空间 / Namespace
{
    public static class Program      // 静态类 / Static Class
    {
        static void Main(string[] args) // 方法 / Method
        {
            int age = 18;             // 声明+赋值 / Declare

            if (age >= 18)               // 分支 / Branch
                Console.WriteLine("成人 Adult");
            else
                Console.WriteLine("未成年 Minor");

            for (int i = 0; i < 3; i++) // 循环 / Loop
                Console.WriteLine(i);

            var sum = Add(1, 2);          // 调用 / Call
            Console.WriteLine(sum);
        }

        static int Add(int a, int b)
        {
            return a + b;             // 返回 / Return
        }
    }
}
  • 引用 / Using：using System; 表示要用到 System 这个库里的东西，比如 Console。
  • 命名空间 / Namespace：把一组类装在一个 “盒子” 里，避免重名。
  • 静态类 / Static Class：代码的组织单位；static 表示不需要 new 实例化。
  • 方法 / Method：类里能执行的动作。Main 是程序入口。
  • 声明 / Declare：告诉编译器有一个名字叫 age，类型是 int。
  • 赋值 / Assignment：用 = 把 18 放进 age。
  • 分支 / Branch：if / else 根据条件做不同事。
  • 循环 / Loop：for / while 反复执行。
  • 调用 / Call：Add(1, 2) 让另一个方法去算结果。
  • 返回 / Return：用 return 把结果交回去。

3 基础数据类型 Basic Data Types

数据类型决定了一个变量能装什么、占多少内存、能做什么运算。

3.1 值类型 Value Types

类型 / Type含义 / Meaning示例 / Example
int整数 / Integer (32 位)int n = 42;
long长整数 / Long (64 位)long big = 10000000000L;
double双精度小数 / Doubledouble pi = 3.14159;

decimal高精度小数 / Decimal (适合金额)decimal price = 9.99m;

3.2 枚举 Enum (值类型 / Value Type)

🏷️

枚举 Enum

把一组有限的取值起有意义的名字，代码更可读。枚举属于值类型，和 int、bool 一样。

enum Color { Red, Green, Blue }

Color c = Color.Green;
Console.WriteLine(c);  // Green
💡

为什么用枚举？ Why Enum?

避免用"魔数"（0, 1, 2）表示状态，名字更易懂、不容易写错。

// 不好：用数字表示状态
int status = 2;  // 2 是什么？

// 好：用枚举更清晰
enum Status { Idle, Running, Done }
Status s = Status.Done; // 一目了然

3.3 数组 Array · 列表 List (集合类型 / Collections)

📦

数组 Array

定长、同类型数据的集合。索引从 0 开始。

int[] nums = { 1, 2, 3 };
Console.WriteLine(nums[0]); // 1
📋

列表 List<T>

可变长度的集合，可随时 Add、Remove。

using System.Collections.Generic;

List<string> names = new();
names.Add("Tom");
names.Add("Lily");

3.4 值类型 vs 引用类型 Value vs Reference

值类型 Value Type

变量直接装数据本身。赋值会复制。例：int / double / bool / struct / enum。

引用类型 Reference Type

变量存的是 地址 / Reference，多个变量可指向同一对象。例：class / string / array / List<T>。

4 语句详解 Statements

方法体里写的每一行 “干活的话”，都是 “语句”。下面分类列出最常用的几种。

4.1 赋值与运算 Assignment & Operators

赋值 Assignment

int a = 10;       // 普通赋值
a += 5;          // a = a + 5
a -= 2;          // a = a - 2
a *= 3;          // a = a * 3
a /= 2;          // a = a / 2
a++;             // 自增 / increment
a--;             // 自减 / decrement

算术 / 比较 / 逻辑 Arithmetic / Compare / Logic

// 算术 Arithmetic
int x = 7 + 3;
int y = 7 % 3;     // 取余 / mod = 1

// 比较 Compare
bool b1 = (x == 10);
bool b2 = (x != y);
bool b3 = (x > y);

// 逻辑 Logic
bool ok = b1 && b2;  // 与 AND
bool any = b1 || b3; // 或 OR
bool no = !b1;        // 非 NOT

4.2 分支语句 Branching Statements

if / else 条件

if (score >= 90)
    Console.WriteLine("A");
else if (score >= 60)
    Console.WriteLine("Pass");
else
    Console.WriteLine("Fail");

switch 多分支

switch (day)
{
    case 1: Console.WriteLine("Mon"); break;
    case 2: Console.WriteLine("Tue"); break;
    default: Console.WriteLine("?"); break;
}

三元 Ternary

string tag = (age >= 18)
    ? "Adult"
    : "Minor";

4.3 循环语句 Looping Statements

for

for (int i = 0; i < 5; i++)
{
    Console.WriteLine(i);
}

while

int n = 3;
while (n > 0)
{
    Console.WriteLine(n);
    n--;
}

do-while

int k = 0;
do
{
    Console.WriteLine(k);
    k++;
} while (k < 3);

foreach

// 先声明并初始化集合 / Declare collection
string[] names = { "Tom", "Lily", "Bob" };

foreach (var name in names)
{
    Console.WriteLine(name);
}
💡 break 跳出循环；continue 进入下一次循环；return 直接结束当前方法。

4.4 调用与返回 Call & Return

// 定义方法 / Define a method
static double Average(double a, double b)
{
    return (a + b) / 2;        // 返回值 / return
}

// 调用方法 / Call a method
double avg = Average(3, 7);    // 5
Console.WriteLine(avg);

5 修饰符 Modifiers

写在类、方法、字段前的关键字，控制 “谁能用”、“能不能改”、“怎样用”。

5.1 访问修饰符 Access Modifiers

关键字 / Keyword含义 / Meaning谁能访问 / Who can access
public公共任何代码 / anywhere
private私有仅本类内部 / only this class
protected受保护本类 + 子类 / class & subclasses
internal程序集内同一个项目 / same assembly
protected internal组合同一项目 或 子类

5.2 行为修饰符 Behavior Modifiers

常用关键字 Common Keywords

⭐ 最常用：不加修饰的普通类（要 new 实例）和 static 静态类（直接用、不 new）。其它修饰符在进阶时再了解。
  • (无修饰) 普通类：要先 new 出对象才能使用，每个对象有自己的数据。
  • static 静态：属于类本身，不需要 new 实例。
  • readonly 只读：构造完之后不能再改。
  • const 常量：编译期固定的常量值。
  • abstract 抽象：必须由子类实现。
  • virtual 虚：可以被子类 override 覆写。
  • override 覆写：在子类中替换父类的虚方法。
  • sealed 封闭：不允许再被继承或覆写。
  • async / await：异步方法，等待耗时操作。

例子 Examples

// ① 普通类：使用前要 new 一个实例 / Normal class: must new an instance
public class Circle
{
    public double Radius;
    public double Area()
    {
        return 3.14159 * Radius * Radius;
    }
}

// ② 静态类：不能 new，直接用 类名.成员 / Static class: use directly
public static class MathTool
{
    public static double Square(double x)
    {
        return x * x;
    }
}

// 调用方法 / How to call them
static void Main()
{
    // 普通类：必须先声明并 new 出对象
    Circle c = new Circle();   // 声明 + new
    c.Radius = 2;
    Console.WriteLine(c.Area());     // 12.566...

    // 静态类：不需要声明，直接 类名.方法
    Console.WriteLine(MathTool.Square(5)); // 25
}

6 注释符号 Comment Symbols

注释不会被运行，但会让代码更易读，也方便和 AI 解释你的意图。

单行注释 // Single-line

// 这是一行注释 This is a comment
int n = 10; // 行尾注释

多行注释 /* Multi-line */

/*
  多行注释
  Multi-line
  comment block
*/

文档注释 /// XML Doc

/// <summary>
/// 求和 / Add two numbers
/// </summary>
public int Add(int a, int b)
    => a + b;
💡 写给 AI 看的小技巧：在棘手代码上方写一行 // TODO: ... 或 // 这里要做什么，然后让 AI 补全或解释，效率更高。
