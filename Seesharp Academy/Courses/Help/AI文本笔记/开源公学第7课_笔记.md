## 🤖 第一部分：AI 智能体开发演示

### 1.1 扣子 (Coze) 平台概述
扣子是字节跳动推出的**低代码/零代码 AI 智能体开发平台**，其特点包括：
* **国内可用：** 无需科学上网，国内用户可直接使用
* **快速开发：** 通过可视化界面即可创建智能体
* **模型集成：** 调用豆包大模型进行语义理解和回复
* **API 支持：** 支持将智能体部署为 API，方便二次开发

### 1.2 智能体开发流程
本节课演示了如何开发一个**根据温湿度数据提供穿衣建议**的智能体，具体步骤如下：
1. **创建智能体**
   * 登录扣子平台，创建新智能体
   * 设定智能体的提示词（Prompt），定义其角色和行为
   * 配置联网搜索功能，使智能体能够获取实时信息
2. **测试智能体**
   * 在平台内测试智能体功能
   * 输入温湿度数据，验证智能体返回的穿衣建议是否合理
3. **部署与 API 获取**
   * 将智能体部署上线
   * 获取 API 接口地址和 API Token（**重要：Token 仅显示一次，需要安全保存**）
4. **API 调用测试**
   * 使用 Python 脚本测试 API 调用
   * 验证智能体响应的正确性
5. **硬件集成**
   * 修改 ESP32 S3 代码，集成智能体 API
   * 实现硬件与 AI 智能体的通信

### 1.3 核心技术组件
* **ESP32 S3：** 乐鑫科技推出的物联网微控制器
  * 支持 WiFi 和 蓝牙连接
  * 适合物联网应用开发
  * **重要限制：** WiFi 仅支持 2.4G 频段（不支持 5G WiFi）
* **DHT11 传感器：** 常用的温湿度传感器
  * 同时测量温度和湿度
  * 成本低廉，接口简单
  * 精度适中，适合教育和小规模应用

## ⚙️ 第二部分：时钟同步技术

### 2.1 同步的基本概念
在测试测量系统中，**同步是确保多设备、多通道数据一致性的关键技术**。没有精确的同步，数据分析将无法保证准确性和可信度。
同步的两大类别：
* **时钟同步：** 保证各设备使用相同的时钟基准
* **触发同步：** 保证各设备在同一时刻开始/停止采集

### 2.2 时钟同步详解
时钟同步分为两种方式：
* **参考时钟同步**
  * **用途：** 多设备频率锁定
  * **原理：** 各设备锁定到同一个参考时钟源，确保频率一致
  * **应用场景：** 多设备协同工作，需要严格频率匹配的场合
* **采样时钟同步**
  * **用途：** 控制 AD 采样时刻
  * **原理：** 所有采集设备使用同一个采样时钟
  * **重要性：** 保证各通道采样时刻严格一致，是相位同步的基础

### 2.3 触发同步详解
触发同步用于控制采集的开始和停止时机，主要类型包括：
| 触发类型 | 说明 |
| :--- | :--- |
| **启动触发** | 触发后开始数据采集 |
| **停止触发** | 触发后停止数据采集 |
| **多卡任意通道触发** | 任意一块卡、任意一个通道都可以触发所有卡 |

**扩展知识：** 多卡任意通道触发技术实现复杂，业界只有少数厂商能够实现，常见于粒子加速器等大科学装置。

### 2.4 同步对象层次
测试测量系统的同步对象按层次划分为：
* **跨机箱同步：** 不同机箱之间的同步
* **板间同步：** 同一机箱内不同板卡之间的同步
* **通道同步：** 同一板卡上不同通道之间的同步
* **板卡功能同步：** AI/AO/DI/DO/Counter 等不同功能模块间的同步

### 2.5 动态信号采集卡 (DSA)
DSA (Dynamic Signal Acquisition) 采集卡具有以下特点：
* **通道相位同步：** 天生支持多通道相位对齐
* **高精度：** 适合振动、噪声等动态信号测量
* **同步性能好：** 内部时钟 and 触发设计保证了良好的同步特性

### 2.6 软件重采样技术
软件重采样是用于**通道间同步补偿**的软件方法：
* **原理：** 通过数字信号处理技术，对采样数据进行重采样处理
* **限制条件：** 信号带宽必须显著低于二分之一采样率
* **重要概念：** 二分之一采样率是奈奎斯特采样定理的核心参数，即采样频率必须不低于信号最高频率的两倍，以避免混叠失真。

### 2.7 实战案例：等角度采样
* **应用背景：** 汽车方向盘转向角与力矩关系测试
* **实现方式：**
  * **光电编码器：** 将方向盘角度转换为脉冲信号
  * **Counter (计数器)：** 测量编码器脉冲
  * **AI 同步：** 计数器与 AI（模拟输入）同步采集
* **目的：** 等角度采样可以在角度域而非时间域分析信号，更精确地反映机械系统的特性。

### 3.2 关键要点总结
* **AI 辅助编程：** AI 辅助工具使用提供参考案例 + DLL 说明文档，可显著提高 AI 编码准确度。
* **精度表述规范：**
  * **基本精度：** 仪器本身的测量精度
  * **附加精度：** 环境温度、气压等变化时增加的误差
* **数字信号处理：** 数字信号处理工具是测试测量公司的稀缺能力。掌握 DSP 技术（FFT 分析、滤波器设计等）是从事测试测量领域的核心竞争力。
* **时频联合分析：**
  * **适用场景：** 非稳态信号（频率变化的启停过程）
  * **优势：** 能够在时域和频域同时分析信号特征。
